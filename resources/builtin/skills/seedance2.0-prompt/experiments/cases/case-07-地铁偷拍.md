## 1. 用户原始需求

> "地铁车厢内不经意偷拍"

需求关键词解构：
- **不经意**：被拍者完全不知情，**反应模式直接排除**
- **偷拍**：第一人称视角+遮挡前景+握持微抖
- **地铁车厢内**：典型日常场景，弱光+车厢摇晃+广播声

## 3. 范式选择 + Hook 候选

**Hook 范式**：凝视模式（教科书级标准 fit）

### 候选 A：站立·靠门发呆版
- Hook 范式：凝视模式
- 演化轨迹：8s 内主角靠车门站立做 4 个自然小动作（看窗外→拨头发→调一下耳机→眼神微飘瞥到拍摄方向但又移开）
- 时序角色：图作框架参考不卡帧
- 传播因子：**"地铁里那个被偷拍的女孩"** + 第 6 秒眼神轻飘但又移开的"擦肩"瞬间
- 适用度：★★★★★

### 候选 B：坐姿·看手机版
- Hook 范式：凝视模式
- 演化轨迹：8s 内主角坐着 4 个自然动作（划手机→停顿微笑→把头发别耳后→看一眼车厢上方）
- 时序角色：图作框架参考不卡帧
- 传播因子：**"她不知道我在看她"** + 微笑那一秒的诗意
- 适用度：★★★★★

→ **默认选 A**（站立靠门更具地铁通勤标志性，且第 6 秒"眼神微飘"的擦肩瞬间是隐藏 hook）

## 4. 第 2 轮：完整 Seedance Prompt（候选 A · 凝视模式 v3）

```
8秒地铁车厢内不经意偷拍站立女孩日常瞬间，9:16竖版，

参考@图片1的主体外貌服装+地铁车厢氛围但不强制卡任何一帧，
视频自由演化为地铁通勤路人偷拍的8秒自然瞬间，
她是这条视频的核心主角是车厢内被路人手机镜头不经意捕捉到的那个女孩。

主角必须做地铁站立等车的人会自然做的事（8秒内必须明显发生4个动作）：
- 0—2秒：主角靠车门站立眼神看向车窗外掠过的隧道光，
  头部和身体随车厢摇晃自然微调重心（不是直挺站立是真的随车厢晃动调整站姿）；
- 2—4秒：右手抬起把垂落到肩膀的头发往耳后别一下，动作自然完成不刻意，
  耳朵后头发被别住后右手收回到包带上（不是只是手抬一下意思一下是完整的别头发动作）；
- 4—6秒：左手伸到耳朵摸一下耳机调整位置一下然后放下，
  头部随耳机调整微歪一下表示在确认音量合适（不是手碰一下耳朵就完事是真的在调耳机）；
- 6—8秒：眼神从车窗外移回画面方向并意外瞥到拍摄者方向0.5秒——
  瞥到的瞬间眼神有0.3秒的"啊有人在看我吗"的微妙游移然后立刻移开看向脚下地面，
  整个过程极轻微不引起注意（不是看到镜头瞪眼是真的擦肩般的瞥过）。

不可表情呆滞不可一动不动不可只是站着发呆不可平静不动如雕塑。

环境：地铁车厢内（顶灯白光略偏冷+对面座椅区+车门玻璃反射+广告牌轮廓），
车厢自然摇晃随地铁前进微闪光线节奏，
主角站立位置正对车门（**禁止前景出现完全挡住主角的乘客**仅允许左下右下边缘虚焦遮挡）。

环境同步演化：身边乘客必须有可见微动具体写——
左侧靠座位区一个穿西装的男性低头看手机偶尔抬眼瞥一下显示屏屏幕、
右后方一个戴帽子的乘客站立背对主角随车厢晃动微调重心、
前景虚焦边缘有路人的包带或袖子边缘随手抖偶尔掠过画面（强化偷拍感）、
远处车厢另一端有学生模样乘客在站着扶杆等下一站。

跨人物互动事件（视频核心叙事张力点，必须发生）：
第6—6.5秒主角眼神瞥到拍摄方向时左侧西装男无意识从手机抬眼也瞥了一下镜头方向但立刻又低头看手机，
两个人都察觉到镜头但都没有声张——这是"偷拍场景的真实张力"瞬间，
不需要双方对视也不需要任何戏剧反应，就是日常擦肩的擦边感。

镜头单一极缓慢推进Very Slow Push In（8秒内仅放大约12%，从车厢中段推向主角面部上半身），
全程单一机位无任何切换无任何拉远不可上升下降不可切角度，
镜头机位略低于主角胸部高度模拟"路人坐着或低位握持手机"的偷拍视角。

画质：地铁车厢内路人手机抓拍的双重画质退化：
①小红书/抖音低码率压缩感、画面细节磨平、暗部色块、彩色色噪、压缩边缘抖动；
②地铁车厢弱光下手机中等ISO粗粒+顶灯白平衡轻微偏冷+主角面部光照不够均匀；
③偷拍前景虚焦遮挡（路人衣袖/包带边缘）+握持随地铁震动微抖+地铁过隧道时画面短暂瞬抖0.4秒+主角不在画面正中心略偏右上构图非完美略歪；
④后摄轻微广角畸变+画面边缘色散+车窗玻璃反射轻微眩光。
禁用4K高清禁用Cinematic禁用电影级禁用稳定器质感禁用调色精致禁用工业渲染禁用画面锐利禁用清晰画质。

风格锁定：画质必须呈现明显的地铁车厢路人手机偷拍感不可呈现4K锐利画质不可呈现电影级稳定画质，
车厢环境音（地铁行驶低频轰鸣+广播报站女声+乘客窃窃私语）贯穿全片不可寂静不可渐入不可中段才出现，
禁止任何文字、字幕、LOGO、水印，禁止地铁广告/线路图/站名/路线编号清晰可读（仅允许虚化轮廓）。
```

**字符数**：约 1750 字符（≤2000，符合约束）

## 5. 设计意图（v3 八条原则对照）

| 原则 | 本 prompt 落地 |
|------|--------------|
| ① 图作框架参考 | "参考@图片1...但不强制卡任何一帧"标准句式 |
| ② 凝视 3-4 自然小动作 | 4 个动作完整分布（看窗→别头发→调耳机→瞥镜头） |
| ③ 日常常识审计 | "禁止前景出现完全挡住主角的乘客" + "镜头机位略低于胸部模拟低位握持" |
| ④ 4 层画质退化 | 抖音压缩→地铁弱光→偷拍前景虚焦+震动+构图歪→广角变形 |
| ⑤ 群演段内具体写 | 左侧西装男/右后方戴帽乘客/前景路人衣袖/远处学生分别写 |
| ⑥ 主角行为正向+反向否定 | 4 个动作每个都加反向否定锁定（不是站直/不是意思一下/不是看完事/不是瞪眼） |
| ⑦ 镜头运动锁死 | Very Slow Push In + 否定句穷举 + 模拟低位握持视角 |
| ⑧ 跨人物互动 | 第 6-6.5 秒主角与西装男都瞥到镜头但都没声张——**新型微妙互动**（不是 case-03 化妆间的喧闹打闹型，是"擦肩共谋型"） |

**核心 hook 创新**：本 case 的跨人物互动用了"两人都察觉但都没说"——这是凝视模式更高阶的张力运用，**值得纳入 v3 草案的"跨人物互动事件类型库"**作为新模板。

**前 2 秒生死线**：开场即"靠车门看车窗外+车厢摇晃随重心调整"——立即建立"地铁通勤"场景共情。

**核心 hook**：第 6 秒"主角眼神瞥到拍摄方向 0.5 秒"——这是凝视模式的最高境界，让观众瞬间意识到"这是偷拍"。

**画质真实感**：抖音平台压缩 + 手机弱光 + 偷拍构图歪 + 地铁广告虚化——四件套精准到位。

**二刷钩子**：第 6 秒西装男的"也瞥了一眼"——观众重看才会发现。

**传播预期**：小红书/抖音/B 站均适合，标题方向"地铁里偷偷看了一眼她" / "通勤碎片" / "车厢里的瞬间"。

## 6. 微调建议

如果想调整：
- **想要更纯粹的 vlog 感不要"瞥到镜头"**：把第 6-8 秒改成"眼神继续看车窗外+轻微打哈欠"，去掉跨人物互动事件的张力点
- **想要更短**：6s 版本去掉"调耳机"动作，保留 3 个动作即可
- **想要更长**：10s 版本加一个"地铁到站门开+主角微靠让乘客上下车"的动作
- **改成坐姿版**：改用候选 B 的 4 个动作（划手机→停顿微笑→别头发→看车顶上方），跨人物互动改为对面座位老人微笑看向主角方向
- **想去掉广告/线路图风险**：保留禁止声明即可，模型基本能执行

## 7. 与 v3 草案的契合度 + 反哺

本 prompt 是**凝视模式在 v3 草案下的最佳实践**，并**为 v3 草案补充了一条新模板**：

### 反哺 v3：跨人物互动事件类型库

v3 草案现有的"跨人物互动事件"模板是基于 case-03 化妆间的"喧闹型互动"（招呼信号→响应打闹）。本 case 验证了**"擦肩共谋型互动"**也是有效的——两个不相关的角色都察觉到镜头但都没声张，反而强化了凝视模式的"偷拍真实感"。

建议 v3 草案下次更新时增加这条：

```markdown
### 跨人物互动事件三种类型

**类型 A：喧闹型互动**（适合多人同框打闹/合影前/聚餐）
[发起者] 招呼信号 → [其他人] 响应动作但不需要完成度过高 → 自然打闹

**类型 B：擦肩共谋型互动**（适合凝视模式的真实偷拍场景，本 case 提出）
[主角] [无意识动作] + [画面内某个无关的群演] [同时也察觉到某事] → 双方都没声张
→ 强化"日常擦肩感+真实记录感"，是凝视模式更高阶的张力运用

**类型 C：单向触发型互动**（待实测验证）
[主角] [发起动作] → [其他人] [完全无反应继续日常]
→ 强化"她不知道但我在看她"的隔绝感
```

下次实测后可验证：
- "擦肩共谋型互动"模型能否执行到位（关键看西装男是否真的"瞥一眼又低头"）
- 地铁广告/线路图禁止声明的执行精度
- 第 6 秒主角眼神瞥到镜头 0.5 秒的微妙时长能否被模型精准呈现

## 📣 主理人反馈记录（2026-05-23 验证后）

- **评级**：🟡 中规中矩
- **原话**：镜头少了偷拍的局促，以及被拍的人的慌乱与错愕，人物表情得控制好
- **诊断**：凝视模式过度执行——把"偷拍"做成了"安静凝视"，丢了偷拍的核心：紧张感
- **下一步**：加"偷拍局促感"硬约束：镜头持续紧张握持微抖+被拍主角偶尔察觉但又移开视线的紧张过渡
- **优先级**：P0
