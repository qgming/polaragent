# 内容策略

这项技能指导你制定「系统化、可持续」的内容策略，让内容不仅「写得好」，更能「传得开、带转化」。核心不是「发内容」，而是「建立内容体系」。

## 何时使用

- 需要制定长期内容规划
- 需要提升内容的搜索排名（SEO）
- 需要优化内容在多平台的分发效果
- 需要分析内容表现并优化策略
- 需要建立内容生产流程

## 核心框架

### 1. 内容金字塔（Content Pyramid）

**金字塔结构（从上到下）：**

```
           ▲
          /顶\     英雄内容（Hero Content）
         /部内\    1-2个/季度
        /分容 \    高投入、高曝光、品牌级
       /───────\
      / 核心内容 \   Hub Content
     /  5-10个/月 \  中投入、持续产出、核心价值
    /  系列化专题 \
   /──────────────\
  /    长尾内容     \  Long-tail Content
 /    20-50个/月    \  低投入、大量产出、精准触达
/  快讯、答疑、片段  \
──────────────────────
```

**三层内容定位：**

1. **英雄内容（Hero）**
   - 目标：品牌曝光、行业影响力
   - 形式：深度报告、品牌视频、大事件
   - 投入：高（数周制作）
   - 传播：广（媒体转载、社交刷屏）

2. **核心内容（Hub）**
   - 目标：持续价值、用户留存
   - 形式：系列文章、教程、专题
   - 投入：中（数天制作）
   - 传播：稳定持续

3. **长尾内容（Long-tail）**
   - 目标：SEO流量、精准触达
   - 形式：问答、快讯、短视频
   - 投入：低（数小时制作）
   - 传播：长尾、积累

### 2. 内容漏斗（Content Funnel）

**用户旅程 → 内容类型：**

```
认知阶段（Awareness）
↓ 吸引注意
内容类型：热点评论、行业报告、病毒视频
目标：让用户知道你存在

兴趣阶段（Interest）
↓ 激发兴趣
内容类型：干货教程、案例分析、对比评测
目标：让用户认为你有价值

考虑阶段（Consideration）
↓ 建立信任
内容类型：产品演示、用户故事、FAQ
目标：让用户相信你能解决问题

决策阶段（Decision）
↓ 促成转化
内容类型：试用指南、价格对比、优惠活动
目标：让用户选择你

忠诚阶段（Loyalty）
↓ 持续互动
内容类型：进阶教程、用户社区、会员福利
目标：让用户持续使用并推荐
```

**不同阶段的内容策略：**

| 阶段 | 内容目标 | 衡量指标 | 分发渠道 |
|-----|---------|---------|---------|
| 认知 | 曝光量 | 浏览量、分享数 | 社交媒体、新闻媒体 |
| 兴趣 | 互动率 | 点赞、评论、收藏 | 公众号、知乎、小红书 |
| 考虑 | 留资率 | 下载、注册、咨询 | 官网、落地页 |
| 决策 | 转化率 | 购买、付费、订阅 | 邮件、客服、直播 |
| 忠诚 | 复购率 | 留存、推荐、UGC | 社群、会员中心 |

### 3. 选题规划（Topic Planning）

**选题来源（5W1H）：**

1. **热点借势（What's trending）**
   - 时事热点：社会新闻、节日节气
   - 行业热点：新产品发布、政策变化
   - 平台热点：话题榜、热搜词

2. **用户痛点（What's painful）**
   - 常见问题（FAQ 挖掘）
   - 负面反馈（客诉、差评）
   - 搜索需求（关键词工具）

3. **竞品分析（What's working）**
   - 竞品爆款内容
   - 行业标杆做法
   - 跨界借鉴

4. **数据驱动（What's performing）**
   - 历史高表现内容
   - 搜索量上升的关键词
   - 用户行为数据

5. **专家洞察（What's insightful）**
   - 行业趋势预测
   - 独家观点
   - 创始人观点

6. **内容空白（What's missing）**
   - 未覆盖的用户需求
   - 未占领的关键词
   - 未探索的内容形式

**选题评估矩阵：**

```markdown
## 选题评估表

| 选题 | 相关性 | 搜索量 | 竞争度 | 时效性 | 成本 | 总分 | 优先级 |
|-----|-------|-------|-------|-------|-----|-----|--------|
| 选题A | 5 | 4 | 2 | 5 | 3 | 19 | 🔴 P1 |
| 选题B | 4 | 5 | 3 | 2 | 4 | 18 | 🟡 P2 |
| 选题C | 3 | 3 | 1 | 1 | 5 | 13 | 🟢 P3 |

**评分标准（1-5分）：**
- 相关性：与品牌/产品的关联度
- 搜索量：目标用户搜索需求大小
- 竞争度：已有内容的竞争程度（越低越好，倒序计分）
- 时效性：内容的时间敏感性
- 成本：制作难度和资源投入（越低越好，倒序计分）

**优先级：**
- 🔴 P1（18-25分）：立即执行
- 🟡 P2（12-17分）：列入计划
- 🟢 P3（5-11分）：储备选题
```

## SEO 优化策略

### 1. 关键词研究

**关键词分类：**

```
品牌词（Brand Keywords）
└─ 公司名、产品名
   例：PolarAgent、Cowork工具

核心关键词（Core Keywords）
└─ 行业主词、产品类别
   例：团队协作工具、AI助手

长尾关键词（Long-tail Keywords）
└─ 具体场景、问题描述
   例：如何提高团队协作效率、适合远程办公的工具
```

**关键词布局：**

- **标题（Title）**：包含核心关键词，控制在 30 字以内
- **描述（Description）**：包含核心+长尾关键词，控制在 120 字以内
- **正文（Body）**：
  - 第一段出现核心关键词
  - 每 200-300 字自然出现一次
  - 用同义词和相关词丰富语义
- **小标题（Headings）**：H2/H3 包含相关关键词
- **图片 Alt**：描述图片内容并包含关键词

### 2. 内容结构优化

**SEO 友好的内容结构：**

```markdown
# 标题：包含核心关键词（H1，唯一）

## 引言（150-200字）
- 开门见山，说明本文解决什么问题
- 包含核心关键词

## 目录（可选，长文推荐）
- 锚点链接，方便跳转

## 主体内容
### 小标题1（H2）
内容...

### 小标题2（H2）
内容...

#### 细分点2.1（H3）
内容...

## 总结
- 概括核心观点
- 包含关键词

## 相关推荐（内链）
- 链接到其他相关文章
- 提升页面权重
```

**内容长度建议：**
- 短文：500-800字（快速解答）
- 中文：1200-2000字（深度讲解）
- 长文：3000-5000字（权威指南）

### 3. 技术SEO要点

**页面优化：**
- ✅ URL 简短且包含关键词（/team-collaboration-tools）
- ✅ 页面加载速度 < 3秒
- ✅ 移动端适配（响应式设计）
- ✅ 使用 HTTPS
- ✅ 清晰的面包屑导航

**元数据优化：**
```html
<!-- 标题 -->
<title>如何提高团队协作效率 | PolarAgent</title>

<!-- 描述 -->
<meta name="description" content="本文介绍5个提高团队协作效率的方法，包括工具选择、流程优化、沟通技巧等，帮助远程团队更高效工作。">

<!-- 关键词（可选） -->
<meta name="keywords" content="团队协作,协作工具,远程办公,效率提升">

<!-- Open Graph（社交分享） -->
<meta property="og:title" content="如何提高团队协作效率">
<meta property="og:description" content="5个方法帮助远程团队更高效工作">
<meta property="og:image" content="https://example.com/cover.jpg">
```

## 效果评估

**内容指标体系：**

```markdown
## 内容表现评估

### 流量指标
- 浏览量（PV）：总阅读次数
- 独立访客（UV）：去重后的用户数
- 来源分布：搜索/社交/直接/外链

### 互动指标
- 点赞数：内容认可度
- 评论数：内容讨论度
- 分享数：内容传播度
- 收藏数：内容价值度

### 转化指标
- 点击率（CTR）：点击/曝光
- 转化率（CVR）：转化/点击
- 停留时长：平均阅读时间
- 跳出率：单页访问占比

### 业务指标
- 新增注册/下载
- 咨询/试用申请
- 实际成交金额
- ROI = 收入 / 投入

### 评估模板
| 内容 | PV | UV | 点赞 | 分享 | 转化 | 评分 |
|-----|----|----|-----|-----|-----|-----|
| 文章A | 5000 | 3200 | 120 | 45 | 8 | ⭐⭐⭐⭐ |
| 文章B | 12000 | 8000 | 350 | 120 | 25 | ⭐⭐⭐⭐⭐ |

**评分标准：**
- ⭐⭐⭐⭐⭐：爆款（复制经验）
- ⭐⭐⭐⭐：优秀（继续优化）
- ⭐⭐⭐：及格（分析原因）
- ⭐⭐：待改进（调整策略）
- ⭐：淘汰（停止投入）
```

## 内容生产流程

**标准化流程：**

```
选题 → 审核 → 创作 → 审稿 → 发布 → 数据追踪 → 复盘优化

1. 选题（Topic Planning）
   - 收集选题（每周五）
   - 评估打分（每周一）
   - 确定计划（排期到人）

2. 创作（Content Creation）
   - 拟定大纲（提交审核）
   - 撰写初稿（按模板）
   - 配图/视频（符合规范）

3. 审稿（Review）
   - 内容审核（事实准确、逻辑清晰）
   - 合规审核（无违规风险）
   - 终审（质量把关）

4. 发布（Publishing）
   - 排期发布（按日历）
   - 多平台分发（一稿多发）
   - 互动回复（及时响应）

5. 复盘（Retrospective）
   - 数据收集（发布后3天）
   - 效果分析（每周五）
   - 优化迭代（应用到下期）
```

## 输出要求

- 内容策略文档使用 `write_file` 保存
- 内容日历用表格或 Mermaid 甘特图展示
- 关键词研究提供具体数据（搜索量、竞争度）
- 平台分发计划包含具体标题和发布时间
- 效果评估用数据支撑，不凭感觉
