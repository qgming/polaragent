```
仙风道骨老者，白发白须垂胸，穿月白道袍缀云纹银线，
手持打神鞭（鞭身金色符文流光），脚踩四不像神兽（龙首虎身熊腰麒麟尾，
通体雪白带金纹）。背景：封神台高台，云海翻涌，封神榜金册在身后悬浮展开，
金色符文如雪花飘落。
```

### 二郎神
```
壮年战神，身材魁梧健硕（胸肌腹肌轮廓清晰，臂膀粗壮），
上身仅护肩甲+胸甲+护腰铠（上臂腹肌侧腰裸露），
额间第三只眼射出金色竖瞳光柱，身披银白战袍，腰间挎三尖两刃戟。
身旁哮天犬巨大威猛（黑色机械感神犬，蓝色项圈等离子光）。
背景：灌江口瀑布战台，银白水花飞溅，天光破云金色光柱。
```

### 妲己
```
狐妖形态美人，穿墨黑丝绒改良妖冶宫装——
低胸 V 领深至胸口锁骨可见，胸部丰满若隐若现；
```

### 5 角色"招牌动作"（聚焦时触发）

这是选角界面最关键的生动感来源，**每个角色被聚焦的 1-1.5 秒内完成一个标志性动作**：

| 角色 | 招牌动作 | 动作特征 |
|:---|:---|:---|
| **姜子牙** | 缓缓举起打神鞭 | 鞭身符文金光流动一闪，白须微动 |
| **二郎神** | 第三只眼骤然睁开 | 金色竖瞳光柱射向前方，哮天犬低吼 |
| **妲己** | 回眸一笑 + 九尾展开 | 九尾从裙下缓缓展开如孔雀开屏，眼瞳化狐金竖瞳 |
| **聞仲** | 挥金鞭劈雷 | 金鞭划弧，紫色雷电从鞭尖炸开，三眼被电光照亮 |
| **紂王** | 举爵邀月 | 青铜酒爵举起，仰头饮尽，冕旒微晃 |

**动作原则**：
- 每个动作必须在 1-1.5 秒内完成（Seedance 单次过渡时长内）
- 动作结束回到 Idle（微微呼吸），等待下一次触发或确认
- 动作的"势"比"形"重要（不需要完美 ref，需要气场）

## UI 跨图一致性策略

选角片要求 5+1 张图的共性 UI 骨架高度一致。**操作流程：**

```
Step 1：先生成 @图片1「姜子牙·常态聚焦图」作为基准主图
         抽 4-5 次择优，确保共性 UI 骨架满意

Step 2：后续 4 张常态聚焦图（二郎神/妲己/聞仲/紂王）的 prompt 开头加：
   "参考 @图片1 的 UI 骨架、缩略图列表位置、LOGO 布局、
    按钮提示、字体风格，仅将以下 6 项换成 XX 对应内容：
    ①LOGO 大字 ②中央立绘 ③焦点框位置（移到第 N 个缩略图）
    ④角色铭牌 ⑤背景场景 ⑥整体色调。"

Step 3：「妲己·最终选中图」基于 @图片3 妲己常态聚焦图增改：
   "参考 @图片3 的完整构图，变化：
    ①焦点框放大并金光爆闪 ②九尾全展 + 眼瞳化狐金
    ③其他 4 个缩略图变半透明灰度 ④缩略图 3 右上角加金色 ✓
    ⑤底部『確認 [A]』按钮按下状态（下陷 + 金光）"

Step 4：视频生成时 5+1 张图作为首尾帧约束
```

## 📌 项目经验（重要·必读）

这份规范的前身是 v1.0 完整版（1600 字）+ 一份 1100 字的「精致度强化补丁」。
实战验证后发现：**prompt 越长越详细，AI 反而越拘束，出图越平庸**。

**GPT Image 2 / NanoBanana 等新一代图片模型有视觉直觉与氛围推理能力**，
关键是给它**"对标参照物 + 氛围感 + 轻度磨损细节"**，
而非"像素数值 + 物理渲染参数 + 技术堆砌"。

✅ **有效**：
- "对标黑神话悟空/潘金莲"（一句锚定胜过千言万语）
- "衬衫有咖啡渍、键盘磨得发光"（生活痕迹让 AI 画出实感）
- "UI 克制不抢戏"（视觉类比胜过像素数值）
- "有飞白有枯笔的江湖体书法"（氛围词 > 技术参数）
- "对标原神/艾尔登法环选角界面"（对选角界面的精准锚定）

❌ **无效（已被证伪）**：
- 像素数值约束（≤45px）→ AI 不理解数值
- Octane/SSS/工业光魔/UE5 堆砌 → 让 AI 画"看起来高级"但其实平庸
- 20+ 条"禁止 X"列表 → AI 把 prompt 当考试题做，失去灵气
- 1100 字补丁追加 → 指令冲突，AI 顾此失彼
- 2.35:1 电影黑边 → 选角/菜单应该是 16:9 满屏实机，只有 CG 过场才加黑边

**所以本规范极简。如果某张图效果不好，先检查"有没有写'对标潘金莲/对标原神选角'"，而不是"还要加多少指令"。**

## 🎬 视频生产原则（v3.0 关键）

Seedance 2.0 **做不到**："UI 像素级不动 15 秒 + 中央立绘瞬间切换"（会变成面部 morph）。

所以选角片采用**分段生成 + 首尾帧约束**：

```
一条 15 秒选角片 = 5 段 3 秒 Seedance 片段 + 后期拼接
每段用"首帧图 → 尾帧图"作为约束
UI 在每 3 秒都被基准图拉回，不会整体漂移
```

**关键技术点**：
- 5+1 张基准图必须先出稳定
- 每段 prompt 只描述"这 3 秒发生的变化"（焦点框移动 + 角色招牌动作）
- 不要在视频 prompt 里重复 UI 细节（首尾帧图自然约束）
