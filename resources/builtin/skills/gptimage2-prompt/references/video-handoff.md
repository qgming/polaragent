# 交付给视频生成

用于把 GPT Image 2 生成的图片资产交给 Seedance 2.0、即梦、图生视频或其他视频模型继续生成短剧/短片。

## 目标

GPT Image 2 负责把“前期资产”做清楚；Seedance 负责把这些资产动起来。图片提示词不需要把所有运动都写完，但要给后续视频留下足够清晰的延展空间。

## 图片资产命名建议

输出提示词时，如果用户在做完整短剧项目，可以顺手建议图片资产用途：

- 主角角色设定图：用于锁定人物外貌、服装和体态。
- 角色单人卡：用于 Seedance `@图片1` 角色一致性引用。
- 双人关系图：用于锁定两人站位、距离和权力关系。
- 场景设定图：用于 Seedance `@图片2` 场景结构引用。
- 首帧图：用于图生视频第一帧。
- 关键帧图：用于转折、动作高潮、情绪爆点。
- 道具设定图：用于物品特写和剧情线索。

## 为视频保留空间

首帧图和关键帧图要避免“画面已经结束”。优先写：

- 动作前摇：手刚碰到门把、车灯刚扫过脸、角色刚抬眼。
- 动作后果：杯子刚摔碎、文件刚落地、灯刚灭一半。
- 视线方向：角色看向画外，为下一镜头留出口。
- 构图余量：人物前方、头顶或画面一侧留出运动空间。
- 物理可延展元素：雨线、烟雾、衣摆、头发、灯光闪烁、车流、窗帘。

## 避免不利于图生视频的图片

- 不要把角色切得太满，缺少运动空间。
- 不要让动作定格在最末端，模型很难继续。
- 不要让强烟雾遮住脸和手。
- 不要在首帧里放太多文字、字幕、海报元素。
- 不要让同一张图出现多个相似角色，容易串脸。
- 不要把场景画成抽象背景，缺少可运动的空间结构。

## 输出给用户时的交接说明

如果用户要完整短剧制作方案，图片提示词后可以补一段简短交接，不要过长：

```text
【视频交接建议】
这张图适合作为 Seedance 图生视频的 @图片1 首帧。视频提示词里不要重复长篇描述人物外貌，只需要写“参考@图片1保持人物外貌、服装和场景一致”，然后补充时长、运镜、动作和情绪变化。
```

## 和 Seedance 的协作

当任务同时涉及生图和视频时：

1. 先用 `gptimage2-prompt` 产出角色、道具、场景、首帧或关键帧图提示词。
2. 再用 `seedance` 把图片编号写进视频提示词，例如 `参考@图片1的人物外貌和服装保持一致`。
3. 视频提示词聚焦动作、运镜、节奏、台词、音效和一致性约束。
