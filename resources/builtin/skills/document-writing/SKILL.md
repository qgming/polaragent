# 文档写作

这项技能指导你撰写、润色、整理各类商务文档，让文档「清晰、专业、可直接使用」。核心不是「写得多」，而是「写得对、写得好」——结构清晰、逻辑连贯、格式规范。

## 何时使用

- 需要撰写工作报告（周报、月报、项目报告、总结汇报）
- 需要编写方案文档（项目方案、活动策划、实施计划）
- 需要制作说明手册（操作手册、用户指南、培训材料）
- 需要起草商务沟通（邮件、通知、公告、会议纪要）
- 需要润色现有文档或整合多人文档

## 文档类型与模板

### 1. 工作报告

**周报/月报模板：**
```markdown
# [时间范围] 工作报告

## 本周/月工作概述
- 核心工作1：完成情况及成果
- 核心工作2：完成情况及成果

## 数据亮点
- 关键指标1：[数值] (环比/同比变化)
- 关键指标2：[数值] (环比/同比变化)

## 遇到的问题与解决
- 问题：具体描述
- 解决方案：采取的措施及结果

## 下周/月计划
- [ ] 计划1：具体内容及预期成果
- [ ] 计划2：具体内容及预期成果

## 需要协助的事项
- 事项1：具体需求及期望支持
```

**项目总结报告模板：**
```markdown
# [项目名称] 总结报告

## 项目概况
- 项目目标：
- 时间周期：YYYY-MM-DD 至 YYYY-MM-DD
- 项目成员：

## 目标达成情况
- ✅ 目标1：达成情况及数据
- ⚠️ 目标2：部分达成，差距原因
- ❌ 目标3：未达成，原因分析

## 关键成果
1. 成果1：具体描述及价值

### 做得好的地方

- 经验2：具体做法

### 需要改进的地方
- 问题1：原因分析及改进建议
- 问题2：原因分析及改进建议

- 建议2
```

## 写作原则

### 结构原则

**金字塔原理：**
- 先结论后细节（先说「是什么」，再说「为什么」）
- 上下对应（每个层次的论点都支撑上层结论）
- 分类清晰（MECE原则：相互独立、完全穷尽）

**层次分明：**
- 一级标题：大章节（# 一、二、三）
- 二级标题：小节（## 1.1、1.2）
- 三级标题：要点（### 1.1.1）
- 不超过4层（过深不易阅读）

### 语言原则

**简洁有力：**
- ❌ 冗长：「鉴于目前的实际情况，我们认为有必要对现有流程进行一定程度的优化」
- ✅ 简洁：「现有流程需要优化」

**准确无歧义：**
- ❌ 模糊：「尽快完成」「大概率」「基本满意」
- ✅ 准确：「6月15日前完成」「80%概率」「满意度4.2/5」

**书面语规范：**
- 口语 → 书面语
  - 「挺好的」→「较好」
  - 「弄清楚」→「明确」
  - 「搞定」→「完成」

### 格式原则

**统一性：**
- 标题编号统一（全用数字或全用中文）
- 日期格式统一（YYYY-MM-DD 或 YYYY年MM月DD日）
- 术语统一（「用户」vs「客户」选一个，全文一致）

**可读性：**
- 善用列表（分点陈述，不写长段落）
- 善用表格（对比数据、任务分配）
- 善用加粗（关键结论、重点提示）
- 善用小标题（每200-300字一个小标题）

## 特殊场景处理

### 多人文档整合

**整合流程：**
1. 通读所有文档，列出各自的核心观点
2. 识别重复内容（保留最精炼的版本）
3. 识别冲突内容（需与作者确认或标注分歧）
4. 统一术语和口径
5. 调整结构使之连贯
6. 统一格式和风格

**常见问题处理：**
- 术语不一致 → 制定术语表，全文替换
- 重复内容 → 合并同类项，删除冗余
- 风格不一致 → 统一为目标读者适合的风格
- 逻辑跳跃 → 补充过渡段落

### 文档润色

**润色检查清单：**
- [ ] 逻辑：论点是否支撑结论？有无跳跃？
- [ ] 语言：有无冗长句子、生僻词、语病？
- [ ] 格式：标题层级、列表编号是否统一？
- [ ] 完整：该有的章节都有吗？有无缺漏？
- [ ] 准确：数据、日期、人名是否正确？

**常见语病：**
- 成分残缺：「经过调研」→「经过调研后发现」
- 搭配不当：「提高质量」→「提高水平」或「保证质量」
- 语序混乱：「不仅他来了，而且他还带了礼物」→「他不仅来了，而且还带了礼物」

## 输出要求

- 默认使用中文，语言专业但不生硬
- 必须使用 `write_file` 工具保存最终文档
- 文档命名规范：`[类型]-[主题]-[日期].md`
- 提供文档结构说明，便于后续修改
- 关键数据和结论用**加粗**突出
