# 商务写作规范

本文档说明商务文档的写作规范，确保文档清晰、专业、易读。

## 基本原则

### 1. 金字塔原理

**结构：先结论后细节**

```
结论（最重要的信息）
├─ 论据1（支撑结论）
│  ├─ 细节1.1
│  └─ 细节1.2
├─ 论据2
│  ├─ 细节2.1
│  └─ 细节2.2
└─ 论据3
```

**示例：**
```
❌ 差的写法（铺垫太长）：
我们调研了市场情况，分析了竞品数据，经过团队讨论，考虑了各种因素...最终建议采用方案A。

✅ 好的写法（开门见山）：
建议采用方案A。理由：1）市场需求最大 2）技术成熟度高 3）成本最优。以下是详细分析...
```

---

### 2. SCQA 结构

**Situation（情境）→ Complication（冲突）→ Question（问题）→ Answer（答案）**

**适用场景：** 问题分析、方案建议

**示例：**
```markdown
## 关于提升用户留存的建议

**S（情境）：**
我们的产品上线3个月，新增用户稳定增长。

**C（冲突）：**
但用户留存率持续下降，从首月的40%降到本月的25%。

**Q（问题）：**
如何提升用户留存率？

**A（答案）：**
建议实施以下3项措施：
1. 优化新手引导（预期提升10%）
2. 增加Push推送（预期提升5%）
3. 建立积分体系（预期提升5%）
```

---

### 3. 5W2H 原则

**What（什么）、Why（为什么）、When（何时）、Where（何地）、Who（谁）、How（如何）、How much（多少）**

**适用场景：** 项目方案、活动策划

**示例：**
```markdown
## XX活动方案

- What：端午节促销活动
- Why：提升GMV，清理库存
- When：6.20-6.22（3天）
- Where：线上商城 + 线下门店
- Who：
  - 负责人：张三
  - 执行团队：市场部5人
- How：满100减30 + 限时秒杀
- How much：预算10万元
```

## 语言规范

### 书面语 vs 口语

```
口语化 → 书面语

✗ 搞定 → ✓ 完成
✗ 弄清楚 → ✓ 明确
✗ 挺好的 → ✓ 较好/良好
✗ 差不多 → ✓ 大致/约
✗ 有点 → ✓ 略微/稍
```

### 精准表达

**避免模糊词汇：**
```
❌ 尽快完成
✅ 6月15日前完成

❌ 大概率会成功
✅ 预计成功率80%

❌ 效果还不错
✅ 转化率提升了15%

❌ 很多用户反馈
✅ 收到86条用户反馈
```

### 简洁有力

**删除冗余：**
```
❌ 鉴于目前的实际情况，我们认为有必要对现有流程进行一定程度的优化
✅ 现有流程需要优化

❌ 在XX方面，我们将会进行积极的推动
✅ 我们将推动XX

❌ 该项目已经基本上接近完成
✅ 该项目已完成90%
```

## 格式规范

### 标题层级

```
# 一级标题（文档标题）
## 二级标题（章节）
### 三级标题（小节）
#### 四级标题（要点）

注意：
- 不超过4级（过深不易阅读）
- 层级清晰（不跳级）
- 编号统一（全用数字或全用中文）
```

### 列表使用

**有序列表（步骤/优先级）：**
```markdown
1. 第一步：准备材料
2. 第二步：提交申请
3. 第三步：等待审核
```

**无序列表（并列关系）：**
```markdown
- 优势1：成本低
- 优势2：周期短
- 优势3：风险小
```

**多级列表：**
```markdown
1. 产品功能
   - 核心功能
     - 用户注册
     - 数据展示
   - 辅助功能
     - 消息通知
     - 帮助文档
```

### 表格使用

**对比数据：**
```markdown
| 方案 | 成本 | 周期 | 风险 | 推荐度 |
|-----|------|-----|------|--------|
| 方案A | 10万 | 3月 | 低 | ⭐⭐⭐⭐⭐ |
| 方案B | 5万 | 6月 | 中 | ⭐⭐⭐ |
| 方案C | 15万 | 1月 | 高 | ⭐⭐ |
```

**任务分配：**
```markdown
| 任务 | 负责人 | 截止时间 | 状态 |
|-----|-------|---------|-----|
| 需求调研 | 张三 | 06-15 | ✅ 已完成 |
| 方案设计 | 李四 | 06-20 | ⏳ 进行中 |
```

## 常用文档模板

### 工作周报

```markdown

## 本周工作概述
[一句话总结本周核心工作]

## 详细工作内容

### 1. [项目/任务名称]
- 工作内容：[具体做了什么]
- 完成情况：[进度/成果]
- 遇到的问题：[问题描述]
- 解决方案：[如何解决]

### 2. [项目/任务名称]
[同上]

## 数据亮点
- 指标1：XX（环比↑10%）
- 指标2：XX（同比↑15%）

## 下周计划
- [ ] 计划1：[具体内容]
- [ ] 计划2：[具体内容]

## 需要协助的事项
- 事项1：[具体需求]
```

## 检查清单

文档完成后自查：

**内容层面：**
- [ ] 结论清晰（开门见山）
- [ ] 逻辑连贯（无跳跃）
- [ ] 论据充分（有数据支撑）
- [ ] 完整性（该有的章节都有）

**语言层面：**
- [ ] 书面语规范（无口语化表达）
- [ ] 精准表达（无模糊词汇）
- [ ] 简洁有力（删除冗余）
- [ ] 无语病（主谓宾完整）

**格式层面：**
- [ ] 标题层级清晰
- [ ] 列表编号统一
- [ ] 表格格式规范
- [ ] 日期格式统一（YYYY-MM-DD）

**数据层面：**
- [ ] 数据准确（经过核对）
- [ ] 数据来源标注
- [ ] 单位统一（元/万元/百万）

---

## 总结

商务写作的核心：
1. **清晰**：让读者快速抓住重点
2. **准确**：用数据和事实说话
3. **简洁**：删除一切冗余
4. **专业**：格式规范，语言得体
