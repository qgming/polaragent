## 各模型详细说明

### Seedance 2.0（字节 · 豆包 / 即梦 / 火山引擎）

- **IP 过滤**：严格。具体 IP 名、可识别角色造型会直接拦截
- **语言**：中英文皆可，中文模型层理解无明显差异
- **时长限制**：
  - 即梦网页版 / 火山引擎 API：单镜头 **4-15s**
  - 豆包 App 消费版：**锁死 5s / 10s** —— 用户在豆包就别承诺 15s
- **反向提示词框**：
  - 火山引擎 API / 即梦网页：有
  - 豆包 App：**不靠谱**—— 消费级豆包 App 里输入框并不可靠地出现，如果用户用豆包 App，把否定写进正向提示词
- **特殊注意**：
  - IP 词过审技巧：可以删除部分标点、换用描述性写法
  - 涉及"能量护盾/脉冲发光"等抽象特效，AI 容易理解为叠加光效而非物理体
  - 建议抽 5-10 次挑最好一版

### Veo 3 / 3.1（Google）

- **IP 过滤**：严格
- **语言**：偏英文，中文描述效果不稳定
- **时长**：每段 8s，可按 7s 步长延长
- **反向提示词框**：✅ 有 —— **只能写纯名词短语**，`no…` / `don't…` 命令式会被拒绝
  - ✅ 正确写法：`blurry, low resolution, watermark`
  - ❌ 错误写法：`no blur, don't add watermark`
- **特殊注意**：IP 过滤不仅查名字，还查"特征组合"—— 描述性写法也要避开可识别的标志性造型

### 可灵 2.x / 3.0（快手）

- **IP 过滤**：中等—— 比 Seedance 宽容，但**一个违禁词就能拒掉整条提示词**
- **语言**：中英文皆可
- **时长**：5-10s（3.0 单条最长约 15s）
- **反向提示词框**：✅ 有 —— 用来压滑步/多指/形变等特定瑕疵
- **特殊注意**：
  - **提交前必须净化身体/接触类用词**："拥抱"→"手臂环绕"、"抚摸"→"手轻触"
  - 物理运动表现好，但细腻互动（如投喂/轻触）容易做飘 —— 写明"手缓慢推过去，罐头停在地面 0.5 米外"
  - IP 相对宽容，使用描述性替代写法即可

### 海螺 / MiniMax

- **IP 过滤**：中等
- **语言**：中英文皆可
- **时长 / 分辨率**：**二选一** —— 1080p 约 6s，768p 约 10s
- **反向提示词框**：✅ 有 —— 建议只针对具体瑕疵少量使用，不要塞太多通用反向词
- **特殊注意**：分辨率和时长的 tradeoff 要提前告知用户

### Wan 2.x（阿里 · 开源）

- **IP 过滤**：较宽松（自部署时；官方平台可能更严）
- **语言**：偏中文 —— 难拍的镜头/首尾帧模式加中文描述效果更好
- **时长**：约 3-8s（新版本约 10-15s）
- **反向提示词框**：✅ 强
- **特殊注意**：自部署才能享受宽松过滤；使用官方平台仍需注意 IP

### Runway Gen-4 / 4.5

- **IP 过滤**：严格
- **语言**：英文
- **时长**：5s 或 10s
- **反向提示词框**：❌ **没有** —— `no X` 写法反而会召唤出 X
- **特殊注意**：
  - **只描述「应该出现什么」**，不写否定句
  - 例：不写"no text overlay"，改写"clean frame, visual storytelling only"

### Pika 2.2 / 2.5

- **IP 过滤**：中等
- **语言**：英文
- **时长**：标准 5s / 10s；Pikaframes 关键帧模式约 25s（非通用功能，仅特定场景）
- **反向提示词框**：2.5 ✅ 有 / 2.2 需在 App 内确认
- **特殊注意**：Pikaframes 不是常规视频生成长度，不要对普通用户推荐

### Sora 2 / 2 Pro（OpenAI）

- **IP 过滤**：**最严格** —— 三层过滤会抓「形似的描述」而非仅名字
- **语言**：英文
- **时长**：Pro 单次最长约 25s
- **反向提示词框**：❌ 没有 —— 把护栏写进**正向提示词**
- **特殊注意**：
  - 避开可识别的特征组合（不仅是名字）
  - 护栏写法：`original character design only, no logos, no text overlay, no morphing geometry`
  - Pro 版支持较长时长，适合分段视频的剪辑后上传

## 反向提示词分流决策

```
目标模型有独立反向提示词输入框吗？
│
├── 有（Seedance/可灵/Veo/海螺/Wan/Pika 2.5）
│   ├── 输入框可信吗？（确认平台）
│   │   ├── 是 → 粘贴标准反向词前缀（见 negative-prompts.md）
│   │   └── 否（如豆包 App）→ 把否定写进正向提示词
│   └── 注意：Veo 和可灵框内禁止 `no…` 命令式
│
├── 没有（Sora / Runway Gen-4）
│   ├── Sora → 把护栏写进正向提示词，用显式 `no ___` 句
│   └── Runway → 只写「应该出现什么」，不要用否定句
│
└── 不确定
    └── 建议用户在 App 里确认，或默认按「有」处理
```
