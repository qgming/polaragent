# 设计审美技能（Taste Skill）

这项技能指导你做出「有品味、不平庸」的设计，防止生成千篇一律的"AI Slop"（平庸的AI生成物）。核心不是「遵循规则」，而是「打破平庸、建立记忆点」。

## 什么是 AI Slop？

**AI Slop 特征（要避免的）：**
- ❌ 过度居中、完全对称的布局（死板）
- ❌ 千篇一律的渐变和圆角（Instagram 滤镜风）
- ❌ 企业蓝 + 安全灰的配色（没有个性）
- ❌ 密集紧凑、没有呼吸感的排版
- ❌ 过度使用 Sans-serif 粗体标题
- ❌ 所有元素等距排列（机械感）
- ❌ 过度使用阴影和毛玻璃效果

**为什么要反对 Slop？**
因为它让所有 AI 生成的设计看起来都一样，没有记忆点，用户无法区分品牌。好的设计应该有个性、有态度、有细节层次。

## 设计三维度（Taste Skill 核心）

**基于 taste-skill v2 的三个可调参数：**

### 1. VARIANCE（设计差异度）0-10
控制布局的不对称程度和视觉张力。

**0-3：传统对称**
- 居中布局
- 等距网格
- 适用：企业官网、政府网站

**4-6：平衡不对称（推荐）**
- 非居中但平衡的布局
- 有主次关系的元素分布
- 适用：大多数商业网站、SaaS 产品

**7-10：高度艺术化**
- 破格布局
- 强烈对比
- 适用：创意工作室、品牌官网、艺术展览

### 2. MOTION（动效强度）0-10
控制页面的动态感和交互反馈。

**0-3：静态/微动**
- 简单淡入淡出
- 适用：文档网站、新闻网站

**4-6：平衡动效（推荐）**
- 滚动视差
- 元素缓动进入
- 悬停反馈
- 适用：营销页面、产品展示

**7-10：沉浸式动效**
- 大量 GSAP 动画
- 滚动触发的复杂编排
- 适用：创意项目、品牌体验站

### 3. DENSITY（信息密度）0-10
控制屏幕上的信息量和留白比例。

**0-3：极简/大量留白**
- Apple 风格
- 一屏只有一个核心信息
- 适用：高端品牌、概念展示

**4-6：平衡密度（推荐）**
- 信息与留白比例得当
- 适用：大多数商业场景

**7-10：信息密集**
- Dashboard 风格
- 一屏展示大量数据
- 适用：后台系统、数据可视化

## 推荐配置

**默认配置（适合大多数场景）：**
```
VARIANCE: 6 (平衡的不对称)
MOTION: 4 (适度动效)
DENSITY: 5 (信息与留白平衡)
```

**创意工作室/品牌官网：**
```
VARIANCE: 8 (艺术化布局)
MOTION: 7 (沉浸式体验)
DENSITY: 4 (留白充足)
```

**SaaS 产品/工具型网站：**
```
VARIANCE: 5 (轻微不对称)
MOTION: 3 (轻量交互)
DENSITY: 6 (信息偏多)
```

**企业官网/政府网站：**
```
VARIANCE: 3 (传统对称)
MOTION: 2 (微动效)
DENSITY: 5 (中等密度)
```

## 设计审查维度

### 1. 布局（Layout）

**✅ 好的布局：**
- 不对称但视觉平衡
- 主次关系清晰（大小、位置、颜色对比）
- 视线引导流畅（F型/Z型阅读路径）
- 留白有意义（分组、强调）

**❌ 避免：**
- 完全居中对齐的所有元素
- 等距分布的卡片网格（改为交错或不等宽）
- 四平八稳的对称布局

**示例改进：**
```
❌ 三栏等宽布局
┌────┬────┬────┐
│ A  │ B  │ C  │
└────┴────┴────┘

✅ 不对称但平衡的布局
┌─────┬──────┐
│  A  │  B   │
│     ├──────┤
│     │  C   │
└─────┴──────┘
```

### 2. 字体（Typography）

**✅ 好的字体排版：**
- 字体层级明确（3-5级：标题/副标题/正文/辅助）
- 字重对比合理（粗细搭配）
- 行高舒适（1.5-1.8倍）
- 字号跳跃合理（1.2-1.5倍渐进）

**❌ 避免：**
- 全部使用 Sans-serif 粗体
- 字号差异过小（标题18px，正文16px）
- 行高过紧（1.2倍）

**推荐字体搭配：**
```
标题：Serif 字体（优雅）或 Sans-serif 粗体（现代）
正文：Sans-serif 常规（易读）
辅助：Sans-serif 细体或小号（层次）

中文推荐：
标题：思源宋体 / 霞鹜文楷
正文：思源黑体 / 苹方
```

### 3. 配色（Color）

**✅ 好的配色：**
- 主色鲜明有个性（不是企业蓝）
- 色彩对比充足（WCAG AA 级）
- 层次分明（60% 主色 + 30% 辅助 + 10% 强调）
- 克制使用渐变（如果用，确保有意义）

**❌ 避免：**
- 蓝色 (#007BFF) + 灰色 (#6C757D) 的安全组合
- 彩虹渐变背景
- 过度使用阴影和毛玻璃

**配色策略：**
```
单色调：一个色相的不同明度
互补色：色轮上相对的颜色（高对比）
类比色：色轮上相邻的颜色（和谐）
三角色：色轮上等距三色（平衡丰富）
```

### 4. 间距（Spacing）

**✅ 好的间距：**
- 基于 4px 或 8px 基础单位
- 相关元素靠近（亲密性原则）
- 不同组之间留白充足
- 不是所有元素都等距

**❌ 避免：**
- 所有元素间距都是 16px
- 卡片之间毫无留白
- 内容紧贴边缘

**间距系统：**
```
4px:  最小间距（图标与文字）
8px:  紧密元素（按钮内边距）
16px: 相关元素（表单字段）
24px: 区块间距（卡片内部分组）
32px: 大区块（页面分段）
48px: 视觉断层（章节分隔）
```

### 5. 交互（Interaction）

**✅ 好的交互：**
- 悬停状态明显但克制
- 动效缓动自然（ease-out）
- 反馈及时（200ms内）
- 加载状态清晰

**❌ 避免：**
- 过度跳动的动画
- 缓慢的淡入淡出（1s+）
- 没有悬停状态

**推荐缓动函数：**
```css
/* 入场动画 */
transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);

/* 出场动画 */
transition: all 0.2s cubic-bezier(0.4, 0, 1, 1);

/* 弹性动画 */
transition: all 0.5s cubic-bezier(0.68, -0.55, 0.265, 1.55);
```

## 反 Slop 检查清单

**设计完成后逐项检查：**

- [ ] **布局**：是否有至少一处不对称但平衡的设计？
- [ ] **字体**：是否有至少3级字体层级（大小+字重）？
- [ ] **配色**：主色是否有个性（不是#007BFF蓝）？
- [ ] **间距**：是否有刻意的留白（不是所有元素等距）？
- [ ] **对比**：视觉焦点是否清晰（有无明确的第一眼看点）？
- [ ] **细节**：是否有至少一处微妙的细节（圆角/阴影/分割线）？
- [ ] **一致性**：元素风格是否统一（不是随意拼凑）？
- [ ] **记忆点**：用户能否在1秒内说出一个视觉特征？

**如果全部打勾，说明设计有品味。**
**如果多项未打勾，说明陷入了 AI Slop。**

## 实战案例

### 案例1：Hero 区域优化

**❌ Slop 版本：**
```
┌────────────────────────────┐
│                            │
│    欢迎来到我们的网站      │
│    [              ]        │  (居中对齐)
│    [    开始使用   ]       │  (大按钮)
│                            │
└────────────────────────────┘
```

**✅ 有品味的版本：**
```
┌────────────────────────────┐
│ 欢迎来到               👋  │  (不对称)
│ 我们的网站                 │  (标题分行)
│                            │
│ [开始使用] 了解更多 →      │  (主次按钮)
└────────────────────────────┘
```

**改进点：**
- 标题左对齐而非居中
- 加入表情符号增加亲和力
- 主次按钮对比明确
- 留白自然分布

### 案例2：卡片设计优化

**❌ Slop 版本：**
```
┌────────────────┐
│ [图片]         │  (阴影重)
│ 标题           │  (居中)
│ 描述文字...    │  (等宽)
│ [按钮]         │  (圆角过大)
└────────────────┘
```

**✅ 有品味的版本：**
```
┌────────────────────┐
│ [图片]             │  (无阴影或轻阴影)
│ 标题               │  (左对齐)
│ 描述文字较短但精炼 │  (不等宽)
│                    │
│ 了解更多 →         │  (文字链接)
└────────────────────┘
```

**改进点：**
- 去除或减轻阴影
- 左对齐而非居中
- 用文字链接代替大按钮
- 适当留白
