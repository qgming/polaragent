# 布局与间距系统

本文档详细说明如何创建有呼吸感、非对称但平衡的布局，避免"机械对齐"的 AI Slop。

## 反 Slop 布局原则

### ❌ 要避免的"AI风格"布局

**1. 完全居中对齐**
```
❌ Slop 布局：
┌────────────────────────────┐
│                            │
│      欢迎来到我们网站      │
│      [大按钮居中]          │
│                            │
└────────────────────────────┘

问题：
- 死板、缺乏动态感
- 像2010年的网站
- 毫无视觉层次
```

**2. 等距网格**
```
❌ Slop 布局：
┌────┬────┬────┬────┐
│ A  │ B  │ C  │ D  │
│    │    │    │    │
└────┴────┴────┴────┘

问题：
- 机械感强，像表格
- 无法突出重点
- 缺乏视觉兴趣
```

**3. 元素紧密堆砌**
```
❌ Slop 布局：
┌──────────────┐
│标题          │
│描述文字描述  │
│[按钮][按钮]  │
│图片图片图片  │
└──────────────┘

问题：
- 透不过气
- 视觉疲劳
- 无法快速扫视
```

### ✅ 有品味的布局策略

**1. 不对称但平衡**
```
✅ 好的布局：
┌─────────────────────────┐
│ 欢迎来到        👋      │
│ 我们的网站              │
│                         │
│ [开始] 了解更多 →       │
│                         │
│         [大图片]        │
└─────────────────────────┘

优点：
- 视觉焦点明确
- 有节奏感
- 留白有意义
```

**2. 黄金比例分割**
```
✅ 好的布局（60/40分割）：
┌───────────┬─────────┐
│           │         │
│   内容    │  侧栏   │
│   60%     │  40%    │
│           │         │
└───────────┴─────────┘

或 F 型布局：
┌─────────────────────┐
│ [导航栏全宽]        │
├──────────┬──────────┤
│          │          │
│  主内容  │   侧边   │
│   2/3    │   1/3    │
│          │          │
└──────────┴──────────┘
```

**3. 充足的呼吸空间**
```
✅ 好的布局：
┌──────────────────────┐
│                      │
│  标题                │
│                      │
│  描述文字，简短但    │
│  精炼，留白充足      │
│                      │
│  [按钮]  了解更多 →  │
│                      │
│                      │
│  [图片]              │
│                      │
└──────────────────────┘

优点：
- 易于扫视
- 视觉层次清晰
- 专业优雅
```

## 间距系统（Spacing System）

### 基础单位

**8px 基础单位系统（推荐）**
```
为什么选择8px？
- 大多数屏幕是偶数分辨率
- 易于计算和缩放
- 符合人眼视觉舒适度
- 主流设计系统的标准（Material Design、Ant Design）

基础单位：
4px:  最小间距（极紧密元素）
8px:  紧密间距（相关元素）
16px: 标准间距（段落、按钮内边距）
24px: 中等间距（卡片内部分组）
32px: 大间距（模块之间）
48px: 超大间距（页面分段）
64px: 巨大间距（章节分隔）
```

**4px 基础单位系统（备选）**
```
适用场景：
- 移动端
- 紧凑型 UI
- 需要更精细控制的场景

基础单位：
4px, 8px, 12px, 16px, 20px, 24px, 32px, 40px, 48px
```

### 间距应用规则

**1. 亲密性原则（Proximity）**
```
相关元素靠近，不相关元素远离

示例：
┌──────────────────┐
│ 标题             │ ← 标题与描述：8px间距
│                  │
│ 描述文字第一段   │
│ 描述文字第二段   │
│                  │ ← 描述与按钮：24px间距
│ [按钮]           │
│                  │
│                  │ ← 按钮与下一区块：48px间距
│ ─────────────    │
│                  │
│ 下一个区块       │
└──────────────────┘

规律：
- 同一组内：4-8px
- 不同组内：16-24px
- 不同区块：32-48px
- 不同章节：64px+
```

**2. 视觉重量决定间距**
```
元素越重要，周围留白越大

示例：
┌───────────────────────────┐
│                           │ ← 顶部留白：64px
│                           │
│    主标题（大号）         │
│                           │ ← 标题下留白：32px
│  副标题（中号）           │
│                           │ ← 副标题下留白：16px
│ 正文（小号）              │
│                           │ ← 正文间距：8px
│ 正文继续                  │
└───────────────────────────┘

规律：
- 大标题周围：32-64px
- 中标题周围：24-32px
- 小标题周围：16-24px
- 正文段落：8-16px
```

**3. 内边距与外边距的平衡**
```
padding（内边距）与 margin（外边距）要协调

按钮示例：
┌─────────────┐
│  [  按钮  ]  │  ← padding: 12px 24px（上下12px，左右24px）
└─────────────┘
     ↓ margin-right: 16px
┌─────────────┐
│  [  按钮  ]  │
└─────────────┘

卡片示例：
┌─────────────────────────┐
│  ← padding: 24px        │
│                         │
│  卡片标题               │
│                         │
│  卡片内容               │
│                         │
│  ← padding: 24px        │
└─────────────────────────┘
     ↓ margin-bottom: 32px
┌─────────────────────────┐
│  下一张卡片             │
└─────────────────────────┘
```

## 网格系统（Grid System）

### 12列网格（标准）

**基础网格：**
```
容器宽度：1200px（桌面端）
列数：12列
列宽：动态（(容器宽度 - 间距) / 12）
间距（gutter）：24px
```

**常用布局组合：**
```
全宽：
┌────────────────────────┐
│  12列                  │
└────────────────────────┘

二分：
┌───────────┬───────────┐
│  6列      │  6列      │
└───────────┴───────────┘

三分：
┌───────┬───────┬───────┐
│  4列  │  4列  │  4列  │
└───────┴───────┴───────┘

主次（2:1）：
┌───────────────┬───────┐
│  8列          │  4列  │
│  主内容       │  侧栏 │
└───────────────┴───────┘

主次（3:1）：
┌─────────────────┬─────┐
│  9列            │ 3列 │
│  主内容         │侧栏 │
└─────────────────┴─────┘
```

### 响应式断点

**标准断点：**
```
xs (Mobile):     <640px   (单列)
sm (Small):      640px+   (2列)
md (Medium):     768px+   (3-4列)
lg (Large):      1024px+  (4-6列)
xl (Extra Large):1280px+  (6-12列)
2xl:             1536px+  (12列)
```

**响应式布局示例：**
```
桌面端（lg+）：
┌─────┬─────┬─────┬─────┐
│  A  │  B  │  C  │  D  │
└─────┴─────┴─────┴─────┘

平板端（md）：
┌─────────┬─────────┐
│    A    │    B    │
├─────────┼─────────┤
│    C    │    D    │
└─────────┴─────────┘

手机端（xs）：
┌─────────────────┐
│        A        │
├─────────────────┤
│        B        │
├─────────────────┤
│        C        │
├─────────────────┤
│        D        │
└─────────────────┘
```

## 留白的艺术

### 留白类型

**1. 微观留白（Micro White Space）**
```
元素内部的小间距

示例：
┌─────────────┐
│  [  按钮  ]  │  ← padding: 上下12px, 左右24px
└─────────────┘

字母间距：
❌ T i g h t (letter-spacing: -0.05em)
✅ Normal (letter-spacing: 0)
✅ R e l a x e d (letter-spacing: 0.05em)

行高：
❌ 行高1.2（太紧）
✅ 行高1.5-1.8（舒适）
```

**2. 宏观留白（Macro White Space）**
```
区块之间的大间距

示例：
┌─────────────────┐
│                 │ ← 顶部留白：80px
│   Hero区        │
│                 │
└─────────────────┘
        ↓ 区块间距：120px
┌─────────────────┐
│   功能介绍      │
└─────────────────┘
        ↓ 区块间距：120px
┌─────────────────┐
│   用户评价      │
└─────────────────┘
```

### 留白比例

**紧凑型 UI（Dashboard）**
```
留白：内容 = 20:80
- 信息密集
- 强调功能
- 效率优先
```

**平衡型 UI（商业网站）**
```
留白：内容 = 40:60
- 信息与留白平衡
- 舒适易读
- 专业感
```

**极简型 UI（品牌官网）**
```
留白：内容 = 60:40
- 留白占主导
- 高端优雅
- 品牌调性强
```

## 实战案例

### 案例1：优化前后对比

**优化前（Slop）：**
```
┌────────────────────────────┐
│       我们的服务           │
├────────┬────────┬──────────┤
│  服务1 │ 服务2  │  服务3   │
│        │        │          │
└────────┴────────┴──────────┘

问题：
- 完全居中，死板
- 等宽等距，机械
- 留白不足，拥挤
```

**优化后（好）：**
```
┌────────────────────────────┐
│                            │
│ 我们的服务                 │
│                            │
│ ┌─────────┐ ┌────┐ ┌────┐ │
│ │         │ │ S2 │ │ S3 │ │
│ │ 服务1   │ │    │ │    │ │
│ │  大卡   │ └────┘ └────┘ │
│ │         │                │
│ └─────────┘                │
│                            │
└────────────────────────────┘

改进点：
- 左对齐标题
- 主卡片突出（大小对比）
- 充足留白（呼吸感）
```

## 检查清单

布局完成后检查：

- [ ] **非居中对齐**（至少有一处非完全居中）
- [ ] **不等宽元素**（避免所有卡片等宽）
- [ ] **基于8px单位**（所有间距是8的倍数）
- [ ] **留白充足**（区块间距≥48px）
- [ ] **视觉层次**（通过大小对比建立层次）
- [ ] **亲密性原则**（相关元素靠近）
- [ ] **响应式适配**（移动端重新布局，不是简单缩小）
- [ ] **易于扫视**（F型或Z型阅读路径）

---

## 总结

有品味的布局要点：
1. **打破完全对称**：不对称但视觉平衡
2. **建立间距系统**：基于8px单位
3. **充足的留白**：呼吸感是专业的标志
4. **大小对比**：通过对比建立层次
5. **响应式思维**：不同设备不同布局
6. **亲密性原则**：相关元素靠近
