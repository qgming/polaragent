# 创建后的校验清单

## 文件结构

- 技能目录位于 `skills/custom/<skill-name>`。
- 目录名只包含小写字母、数字、连字符。
- 根目录存在 `SKILL.md`。
- 如果存在 `references/`、`scripts/`、`assets/`，每个目录至少有一个有用文件。

## SKILL.md

- 文件开头是 YAML frontmatter：

```yaml
---
name: skill-name
description: ...
---
```

- `name` 与目录名完全一致。
- `description` 同时说明能力和触发场景。
- 正文包含实际工作流程，而不是营销介绍。
- 正文引用了需要按需读取的 reference 文件。
- 没有大段复制外部资料或网页原文。

## 可用性

- 使用 `list_directory` 检查目标目录。
- 使用 `read_file` 读取 `SKILL.md`，确认 frontmatter 和正文无明显格式问题。
- 如果写了脚本，至少说明运行方式；能运行时应实际运行一次代表性命令。
- 如果技能依赖网络调研，保留来源链接和日期，说明哪些内容可能随时间变化。

## 交付说明

最终回复用户时说明：

- 创建了哪个技能。
- 技能目录在哪里。
- 包含哪些关键文件。
- 这个技能适合哪些触发场景。
- 是否有未完成的前提，例如需要用户补充模板、API Key、样例文件。
