# 数据分析

这项技能指导你把一份数据变成**可信、可用于决策**的洞察。核心不是「算得出数字」，而是「数字站得住脚」——先确认数据本身可靠，再分析，最后诚实说明结论的适用范围与局限。

## 何时使用

- 用户给了一份数据（销售、用量、问卷、财务、运营指标等）并希望理解它
- 需要判断趋势、对比差异、找异常、做相关分析
- 需要把数据整理成图表用于汇报或决策
- 需要评估「这份数据能不能用来回答某个问题」

## 总体流程（务必按顺序）

1. **对齐目标** —— 先弄清用户要用数据回答什么问题、支撑什么决策。目标不清就先问，不要急着算。
2. **评估数据质量与可用性** —— 在分析前先体检数据（见下方「数据质量六维度」）。数据不可靠时，任何分析都是空中楼阁。
3. **探索性分析（EDA）** —— 先看分布、范围、异常，理解数据长什么样，再决定用什么方法。
4. **针对性分析** —— 围绕目标做统计/对比/趋势/分群，而不是把所有指标都算一遍。
5. **得出洞察并交付** —— 区分「数据说明了什么」与「因此建议怎么做」，标明前提与局限。

## 方法论一：如何确定数据准确、可用

分析前用「数据质量六维度」给数据做体检。这是被广泛采用的行业框架（最早由 Wang & Strong 1996 年提出，现普遍收敛为 6–10 个维度）。

| 维度 | 含义 | 怎么检查 |
| --- | --- | --- |
| **准确性 Accuracy** | 数据是否如实反映现实 | 抽样比对权威来源；找明显不合常理的值（如年龄 999、负的销售额） |
| **完整性 Completeness** | 该有的记录与字段是否齐全 | 统计缺失率（如「95% 记录字段齐全」）；关注关键字段的空值 |
| **一致性 Consistency** | 同一数据在不同来源/系统是否一致、无矛盾 | 跨表核对同一实体；检查格式（日期、单位）、口径是否统一 |
| **时效性 Timeliness** | 数据是否够新、覆盖到所需时间 | 看最后更新时间、数据截止日期；判断是否过时 |
| **有效性 Validity** | 取值是否落在合法范围/格式 | 用业务规则校验（取值范围、枚举值、格式规则） |
| **唯一性 Uniqueness** | 是否有重复记录 | 按主键/关键字段去重，统计重复比例 |

**实操检查清单（分析前必做）：**

- 数据从哪来？怎么采集的？谁负责？采集方式本身有没有偏差？
- 必填字段有多少空值？空值是「真的没有」还是「未记录」？该删除还是补全？
- 有没有重复行、明显异常值（离群点）？异常是真实情况还是录入错误？
- 字段口径是否统一（如「销售额」含不含税、单位是元还是万元、日期格式是否一致）？
- 时间范围是否覆盖要回答的问题？是否存在「只保留了存活样本」的缺口（见幸存者偏差）？
- 样本是否能代表你想推断的总体？（见抽样偏差）

> 体检不达标时：明确告诉用户哪些维度有问题、影响有多大，能补救就说明补救方式（清洗/补全/换口径），不能补救就说明结论会受什么限制——**绝不在脏数据上硬下结论**。

## 方法论二：避免常见统计陷阱

这些陷阱的共同点是「对看不见的东西下了结论」——忽略了混淆变量、缺失样本或不具代表性的样本。

- **相关 ≠ 因果**：两个变量一起变化，不代表一个导致另一个，常有第三个混淆因素（如气温同时推高冰淇淋销量与溺水数）。要确认因果，需控制变量或对照实验。
- **抽样偏差**：样本不能代表总体，结论就不可推广。先问「这批数据是怎么选出来的」。
- **幸存者偏差**：只看「活下来/留下来」的样本会误导（二战只检查返航飞机的弹孔 → 该加固的恰恰是没弹孔的部位）。要追问「哪些数据没出现在表里」。
- **辛普森悖论**：分组内的趋势在合并后可能反转（伯克利招生案）。涉及分组数据时，先分层看再合并。
- **数据钓鱼 / 多重比较**：在同一份数据里反复试到「显著」结果不可信。应先立假设、再用数据验证，别用同一份数据既造假设又验假设。

详见 [references/STATISTICAL_PITFALLS.md](references/STATISTICAL_PITFALLS.md)。

## 方法论三：分析与可视化要点

- **EDA 先行**：先看每个变量的分布（直方图）、范围、缺失与离群，再看变量间关系（散点图、相关系数），最后才建模/下结论。
- **选对图表**：趋势用折线；分类对比用柱状；占比用饼图（类别 ≤6–7 个）；关系用散点；分布用直方图。
- **图表要诚实**：坐标轴从合理基准起（柱状图一般从 0 起）、标清单位与口径、避免误导性截断。
- **报告结构**：先给关键结论（≤3 条）→ 支撑数据与方法 → 前提与局限 → 可执行建议。
- **可追溯**：每个关键数字标明来自哪份数据、哪个时间范围、用了什么口径，方便用户复核。

详见 [references/EDA_AND_CHARTS.md](references/EDA_AND_CHARTS.md)。

## 输出要求

- 默认使用中文，用用户能懂的语言解释统计含义，不堆术语。
- 先结论后依据；结论与建议分开陈述。
- 数据不足或质量有问题时如实说明，不强行下结论。
- 涉及数字一律标注来源与口径。
