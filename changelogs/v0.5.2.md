# 更新日志

## [0.5.2] - 2026-07-10

本次 0.5.2 是一次围绕“协作模式重构”和“工具执行体验”的版本更新。相比 0.5.1，PolarAgent 将原先独立的团队协作体系收敛为更轻、更可控的子代理委派工作流：主助手仍负责理解用户目标和最终交付，遇到调研、代码审查、方案对比、测试验证等复杂子任务时，可调用其它助手或临时创建专家子代理来完成。与此同时，本版本补强了工具执行过程展示、后台任务、项目会话读取、工具页能力标签和运行时队列能力。

### 🔄 协作模式重构

- **移除独立团队协作入口**：下线原侧边栏与扩展导航中的“团队”页面，主界面从“对话 / 项目 / 团队”三段切换收敛为“对话 / 项目”两段切换
- **移除团队会话页面**：删除 TeamPage、TeamChatPage 等团队专用页面，不再维护独立的团队聊天视图
- **移除团队编辑与监控组件**：删除团队编辑弹窗、团队操作菜单、团队监控面板、团队列表等 UI 组件
- **移除团队运行时**：删除团队 Agent 编排、团队成员轮转、团队投票、团队流程控制等旧运行时逻辑
- **移除团队专用工具**：下线 `control_team_flow`、`request_team_vote`、`cast_team_vote` 等团队协作工具
- **移除团队会话存储链路**：删除团队会话 store、团队监控 store、团队面板 store、团队 session 仓库和团队附件/成员/转录等工具模块
- **移除团队语言包与教程入口**：删除中英文 `team.json` 语言包，移除教程中的团队协作章节入口与相关文案
- **配置结构瘦身**：移除配置类型中的团队定义，减少初始化、IPC、preload 和应用启动流程里的团队相关分支
- **首页交互简化**：新对话首页移除“通用 / 团队”类型选择，只保留助手选择、权限模式、知识库、技能和附件等核心入口

### 🧑‍💼 子代理委派

- **新增 `delegate_task` 工具**：主助手可调用另一个助手作为子代理完成明确子任务，适合调研、代码审查、测试验证、方案对比、文案润色和专业判断
- **子代理替代旧团队协作**：复杂任务不再依赖固定团队队列，而是由主助手按需委派，主助手整合子代理结果后向用户交付最终答复
- **子代理上下文隔离**：子代理拥有独立 sessionId，避免和主会话历史混写，同时继承工作目录、权限模式、知识库和项目上下文
- **防止递归委派**：子代理上下文中禁止再次调用 `delegate_task`，避免无限嵌套和失控协作链路
- **内置助手提示词更新**：默认助手 Cowork 与调研专家的系统提示从“团队协作”改为“子代理协作”，引导模型按新工作流执行
- **子代理结果可追踪**：子代理完成后返回 agentId、agentName、childSessionId 和结果文本，便于主助手整合与界面展示

### 🧑‍🤝‍🧑 子代理选择增强

- **新增 `list_agents` 工具**：可列出当前用户已安装及内置助手，包含 ID、名称、类型、介绍、分类与标签，便于主助手选择合适子代理
- **支持显式选择已安装助手**：`delegate_task` 支持 `agentId` / `agentName`，可在读取助手清单后调用指定助手
- **支持临时子代理**：`delegate_task` 支持 `temporaryAgentName` 与 `temporarySystemPrompt`，主助手可按任务现场创建“代码审查专家”“资料调研员”等临时角色
- **临时子代理继承主会话能力**：临时子代理会继承当前模型、工作目录、权限模式、知识库、项目上下文与技能配置，只替换专门的系统提示词
- **默认子代理固定为 `default/Cowork`**：未显式指定目标、也未创建临时子代理时，只会调用 `default` 或名称为 `Cowork` 的默认助手，不再自动选择助手列表第一项
- **找不到目标时提示更明确**：指定助手不存在时会提示先调用 `list_agents`；默认助手不存在时会提示缺少 `default/Cowork`，并建议显式选择助手或创建临时子代理
- **真实子代理名称展示**：调用过程中和完成后，页面显示实际子代理名称，例如“正在调用子代理 Cowork...”或“子代理 代码审查专家 正在执行任务...”
- **历史消息名称恢复**：历史会话中的子代理工具步骤会从 details 中恢复 `agentName`，继续显示真实子代理名称

### 📁 项目会话上下文

- **新增 `list_project_conversations` 工具**：项目会话中可列出同一项目内的其他历史会话，返回标题、ID 和更新时间
- **新增 `read_project_conversation` 工具**：可按会话 ID 读取同一项目内历史会话的最近消息，用于补充上下文、查找之前的结论或约定
- **项目上下文进入工具运行时**：Agent 工具上下文新增 `projectId`，项目专用工具只在项目会话中装配
- **标题索引用于项目检索**：项目会话列表基于 title index 过滤与排序，默认排除当前会话，支持关键词查询与数量限制
- **项目会话视图体验修正**：从侧边栏打开项目对话时会正确切回聊天页，项目内新建对话后也会立即进入对应会话

### 🧰 工具执行体验优化

- **新增工具进度基础设施**：新增统一的工具进度辅助模块，支持阶段摘要、进度更新、取消检查与耗时统计
- **工具执行过程可见**：网络搜索、网页读取、文件操作、知识库检索、图片生成、音频处理、Office 文档、MCP 等工具在执行中可持续回传阶段信息
- **工具结果摘要增强**：工具步骤面板现在能显示更具体的执行结果，例如搜索结果数量、网页标题、知识库命中数、生成图片数量、文件名、MCP 服务与工具名、后台任务状态等
- **执行耗时展示**：支持在工具结果摘要中追加毫秒/秒级耗时，方便判断慢步骤与排查性能问题
- **取消能力补强**：多类工具接入 `AbortSignal` 检查，取消运行时能更及时地停止后续处理
- **运行中状态更准确**：工具有中间更新但尚未完成时，会保持 running 状态，避免过早显示为完成
- **MCP 展示优化**：MCP 工具结果摘要显示为 `serverName · toolName`，更容易识别具体调用来源

### ⏳ 后台任务

- **新增 `start_background_task` 工具**：可将耗时较长、可异步完成的任务放到独立后台 Agent 会话执行，并立即返回 `jobId`
- **新增 `list_background_tasks` 工具**：可查看当前应用内后台任务列表，包括任务 ID、名称、状态、创建时间与完成时间
- **新增 `get_background_task` 工具**：可查询后台任务的状态、执行事件、错误信息和最终结果
- **新增 `cancel_background_task` 工具**：可取消仍在运行的后台任务，并同步中止对应 Agent harness
- **后台任务独立运行**：后台任务使用独立会话与 harness，避免主会话工具调用被长任务阻塞
- **后台执行事件记录**：后台任务会记录启动、工具开始、工具进度、工具完成、失败和完成等事件，便于后续追踪
- **主对话可继续工作**：长任务启动后主对话可以继续回复用户、调用其他工具或处理新的问题，再通过任务查询工具取回结果

### 🧭 工具页与能力元数据

- **新增工具能力元数据**：内置工具增加统一 `capabilities` 描述，覆盖是否支持进度、取消、后台运行、预估耗时和结果展示类型
- **工具页新增能力标签**：工具列表会显示“进度”“可取消”“后台”“中耗时”“长耗时”“产物”“Widget”“音频”等标签
- **新增子代理协作分组**：工具页新增“子代理协作”分组，用于展示 `list_agents` 和 `delegate_task`
- **新增项目工具分组**：工具页新增“项目”分组，用于展示项目会话列表与读取工具
- **移除团队协作分组**：工具页不再显示团队协作工具分组，避免与新的子代理工作流重复
- **中英文国际化补齐**：新增子代理、后台任务、项目会话、工具能力标签等相关中英文文案

### 🔄 运行时与并发能力

- **新增 follow-up 队列能力**：AgentManager 暴露 `followUpThread`，可向当前运行中的线程追加后续消息
- **新增 next-turn 队列能力**：AgentManager 暴露 `nextTurnThread`，支持把消息排入下一轮 prompt 前处理
- **导出线程追加接口**：新增 `followUpAgentThread` 与 `nextTurnAgentThread`，为“工具执行中继续接收用户输入”提供基础能力
- **子代理系统提示可覆盖**：AgentManager 支持对子代理传入专属 `agentName` 与 `systemPrompt`，为临时子代理提供运行时配置入口
- **运行时签名更准确**：harness 缓存签名纳入子代理名称、临时系统提示、项目 ID 与工作目录信息，避免不同上下文复用错误运行时
- **工具上下文更聚焦**：ToolContext 从团队上下文改为子代理、项目、后台任务等更细粒度的上下文字段

### 🐛 修复与行为调整

- **修复默认子代理误选问题**：未指定目标时不再使用助手列表第一项作为隐式默认值
- **修复子代理显示泛化问题**：子代理工具执行中和完成后都显示真实名称，而不是固定展示工具通用标签
- **修复子代理完成态解析不足**：消息解析器补充 `delegate_task` 的结果摘要恢复逻辑
- **修复项目对话打开状态**：选择项目会话或新建项目会话后，主区域会正确进入聊天页
- **修复团队相关残留入口**：移除标题栏、全局搜索、首页、导航、教程、设置、语言包中仍引用团队能力的入口与文案
- **后台任务失败兜底**：后台任务执行异常时会记录失败状态、完成时间和错误消息，而不是静默丢失
- **工具错误摘要更准确**：子代理失败时会显示“子代理 X 执行失败”，便于在步骤面板中快速定位
