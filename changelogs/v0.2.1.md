# 更新日志

## [0.2.1] - 2026-06-16

### 🎉 新功能

#### 横纵分析技能

- **hv-analysis 技能**：新增横纵分析（Horizontal-Vertical Analysis）内置技能，支持对产品、公司、概念、技术或人物进行系统性双轴深度分析，采用 HTML→create_office_document 完整流程生成 PDF，无需外部 Python 脚本依赖

#### PDF 生成器风格扩展（4→14 种）

- 新增 10 种专业风格：academic（学术论文）、minimal（极简）、magazine（杂志叙事）、report（企业报告）、bento（信息图卡片）、letter（信函）、classic（经典档案）、tech（技术文档）、notebook（创意手帐）、modern（现代简约）
- 每种风格配备独立 HTML 模板文件 + CSS 调色板 + 自动选择逻辑
- 更新技能文档和模板指南，含 14 种风格对比表

#### PPT 生成器风格扩展（12→20 种）

- 新增 8 种风格：gradient（渐变炫彩）、handdrawn（手绘风）、tech（科技感）、vintage（复古）、cyberpunk（赛博朋克）、japanese（和风）、korean（韩系简洁）、nature（自然清新）
- 纯 CSS 变量方案，不修改 JS 逻辑，通过 data-ppt-style 属性驱动
- 更新技能文档和模板指南，含 20 种风格对比表

#### 模型服务卡片显式保存

- 模型服务卡片（ProviderCard）展开时底部新增保存栏，含绿色指示灯 +「修改已自动保存」提示
- 右侧「保存」按钮可手动触发保存，状态联动 saving/saved/error
- 保留 onBlur 自动保存逻辑不变，与图片模式、音频设置等面板保持一致设计风格

### 🐛 修复

- **YAML 描述解析器**：parseSimpleYaml() 增加 literal block scalar（|）语法支持，正确处理 `description: |` 多行格式，修复 hv-analysis 等技能描述在前端显示为 `|` 的问题
- **HTML 预览系统浏览器打开**：openInSystemBrowser 改用 `shell.openPath` 直接打开文件路径，替代 `file://` URL + `shell.openExternal` 方案，修复 Windows 上因 file:// 协议处理不当导致「系统找不到指定文件」的错误
- **LLM 429 限流自动重试**：新增 with429Retry 包装函数，对 chatCompletion 和 chatCompletionStream 添加递增间隔重试（1s→3s→5s→10s→30s），最多自动重试 5 次，非 429 错误立即透传，全程用户无感知
