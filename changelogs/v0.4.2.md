# 更新日志

## [0.4.2] - 2026-07-01

### 🎉 新功能

#### 统一 LLM 调用架构

- **新增 `callLlm()` 统一调用工具**：所有轻量级 LLM 调用（标题生成、目标评估、记忆捕获、工具权限审查、语音文字整理）统一使用 pi-ai 的 `streamSimple` API
- **支持所有 Provider 类型**：不再依赖 Electron IPC 的 `/chat/completions` 端点，现在支持 OpenAI、Anthropic、Google 等所有 Provider 类型
- **API 配置统一**：所有 LLM 调用直接使用设置中的模型配置（API key、base URL、model），不再出现 "No API key for provider: openai" 错误

### 🔧 优化

#### 会话存储优化

- **修复会话路径编码问题**：会话仓库不再把绝对路径编码成冗长目录名（如 `--C--Users-...`），改用简洁的 `.` 作为 cwd，编码后为 `--.--`
- **兼容性保留**：旧会话仍可通过 `repo.list()` 找到并正确打开，新会话存储在简洁的目录结构下

#### 代码质量改进

- **移除私有属性访问**：`compaction.ts` 不再通过 `(harness as unknown as).session` 访问 `AgentHarness` 私有属性，`AgentHarness` 0.80 内置自动压缩机制
- **修复 `disposeThread` 参数错误**：`cleanupSessionResources` 现在正确接收 sessionId 而非 agentId
- **清理回调去重**：新增 `registeredCleanupSessions` Set，避免每次创建 harness 都注册重复的清理回调
- **删除冗余代码**：`selectThinkingLevel` 末分支冗余判断已移除
- **修复无效短路**：`updateStep` 中 `Array.prototype.map()` 始终返回新数组的问题已改用 `.some()` 先检测
