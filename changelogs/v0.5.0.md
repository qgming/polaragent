# 更新日志

## [0.5.0] - 2026-07-03

本次 0.5.0 是一次重大特性更新，相比 0.4.3 新增 6 个提交、覆盖约 9000 行代码变更。围绕三条主线能力展开：**后台定时任务**、**对话内 Widget 渲染**、**自定义 Skill 写入**；同时补强了四级权限体系、内置工具的全面优化（27 项功能增强），以及会话统计、记忆管理、LLM 非流式调用重构等多项稳定性提升。

### 🎉 新功能

#### 定时任务（Schedule）—— 可计划执行的 Agent 能力

- **新增 `schedule_task` 工具**：支持 `at`（一次性）、`every`（固定间隔）、`cron`（标准 Cron 表达式）三种调度方式
- **四级权限收口**：后台任务仅允许 `readonly` / `safe` / `ai_review` 三种权限模式，禁止无人监督任务使用 `full` 权限，避免越权
- **独立的 Schedule UI**：新增 `SchedulePage` 与 `ScheduleEditorModal`，支持立即执行、任务启停、日志查看和下次执行时间计算
- **独立会话仓库**：为定时任务引入独立的 `schedule` 会话仓库与会话生命周期，避免与普通对话、团队对话混用
- **调度运行时**：新增调度运行时、Cron 解析、预设与提示词模块
- **国际化文案**：新增 `schedule.json`（中英文 267+ 条文案），并将页面接入应用导航与初始化流程
- **单元测试**：新增 `cron.test.ts`、`prompt.test.ts`、`runtime-ids.test.ts`，覆盖 Cron 解析、提示词生成与运行时 ID 关键逻辑

#### 对话内 Widget 渲染链路

- **新增 `render_widget` 工具**：支持通过内联 HTML 或 skills 目录下的 `.html` 模板生成 Widget
- **WidgetSandbox + patch-engine**：支持 `replace`（整页替换）和 `patch`（局部更新）两种更新模式，可在沙箱中安全渲染交互式卡片
- **消息解析贯通**：消息解析与渲染阶段识别 `widget` 类型 segment，在聊天流中直接渲染交互式 Widget
- **状态合并**：同一 widgetId 仅展示最新状态，减少重复渲染
- **可读摘要**：Widget 工具结果补充可读摘要，便于在步骤面板与历史消息中查看

#### 自定义 Skill 写入与删除能力

- **新增 `write_skill` 工具**：支持 `create` / `edit` / `patch` / `delete` 四类操作
- **Electron IPC 写入通道**：新增 `skills:write-skill`、`skills:patch-skill`、`skills:delete-skill` 三个通道
- **路径与格式校验**：技能名称、frontmatter、路径范围均做校验，避免非法路径与格式错误
- **自动备份**：写入前自动备份 `SKILL.md`，保留最多 10 个版本；删除时先备份整个技能目录
- **自动刷新**：skills store 在写入、补丁、删除后自动刷新技能列表

#### 四级权限体系

- **新增 `safe` 权限模式**：形成 `readonly` / `safe` / `ai_review` / `full` 四级权限体系
- **安全中间件**：`security.cjs` 在主进程侧实现文件路径与命令执行校验
- **双防线同步**：前端权限切换通过 IPC 同步到主进程，确保第二道防线生效

#### Computer Use 增强（Windows 桌面控制）

- **失效重定位增强**：从单次重试改为最多 3 轮递增深度重试，失败时返回 top-3 候选元素列表供 AI 选择
- **按键系统改用 SendInput API**：替代已弃用的 `SendKeys::SendWait`，支持 Windows 键，更可靠的硬件级键码注入
- **截图优化**：支持窗口级/区域截图（`screenshotTarget`），支持 JPEG 格式与质量参数（`screenshotFormat` / `screenshotQuality`）
- **延迟模式可配**：所有操作类工具支持 `delayMode` 参数（`fast` / `stable` / `slow`），11 处硬编码延迟提取为可配置项
- **scroll 使用 UIA ScrollPattern**：提供 `elementId` 时优先使用 `ScrollPattern`，失败回退 `mouse_event`
- **UIA CacheRequest 批量预取**：批量预取常用属性（Name / AutomationId / ClassName 等），减少跨进程调用
- **find 惰性搜索**：DFS 遍历时同步匹配，命中 `maxResults` 即停止，避免构建完整 1200 节点 UIA 树

#### Browser Use 增强（Chrome 浏览器控制）

- **页面就绪检测**：`snapshot` / `click` / `fill` 前自动等待 `document.readyState=complete` + 无 pending 资源 + DOM 静默
- **`@e` 引用容错增强**：操作失败时自动 re-snapshot 并按 selector/text 重新匹配元素，最多重试 1 次
- **CDP 页面导航检测**：`click` / `fill` 前后对比 URL / readyState，检测到导航后等待新页面就绪
- **新增 `browser_drag` 工具**：通过 CDP 注入 JS 模拟 dragstart / dragover / drop 事件序列
- **新增 `browser_upload` 工具**：通过 CDP `DOM.setFileInputFiles` 实现文件上传
- **`browser_click` 增强**：新增 `action` 参数支持 `dblclick` / `contextmenu` / `mousedown` / `mouseup`
- **`browser_fill` 增强**：新增 `selectBy` 参数支持 `<select>` 元素按 `value` / `text` / `index` 选择

#### 文件操作增强

- **新增 `move_file` 工具**：支持重命名和跨目录移动，跨分区时自动回退为复制+删除
- **新增 `copy_file` 工具**：支持文件与目录递归复制
- **新增 `search_files` 工具**：内置轻量 Glob 匹配（支持 `*` / `**` / `?` / `{a,b}`），默认最多返回 100 条
- **`write_file` 新增 `append` 参数**：支持追加写入模式
- **`edit_file` 新增 `regex` / `regexFlags` 参数**：支持正则表达式替换
- **符号链接循环检测**：`search_files` 使用 `visited` Set + realpath 防止无限递归

#### 会话统计（Session Stats）

- **新增 SessionStatsPopover 组件**：导航栏新增「会话统计」入口，聚合展示当前会话关键指标
- **Token 使用度量**：聊天与团队消息类型新增 `tokenUsage` 字段，统计各轮 token 消耗
- **自动记忆抽取**：基于 token 阈值触发自动记忆抽取，无需手动调用

#### 其他新增能力

- **Agent 重试机制**：失败后自动重试（最多 3 次，指数退避），UI 同步展示重试状态（`onRetry` 回调）
- **JSON 输出模式（`jsonMode`）**：支持结构化响应
- **投票并行化**：`team-vote` 从串行改为 `Promise.all` 并行执行
- **媒体懒加载组件 `LazyMediaLoader`**：聊天中媒体资源按需加载
- **MCP 工具 JSON Schema 校验**：调用前验证必填 / 类型 / 枚举 / 长度 / 范围，支持 string↔number 自动转换
- **`update_todos` 增量更新**：新增 `action` 参数支持 `add` / `update` / `delete` / `complete`，不再要求每次传完整列表
- **`ask_user` 超时机制**：新增 `timeout` 参数（1-600 秒），超时后返回友好提示而非永久挂起

### 🔐 安全改进

- **禁止渲染层升权**：明确安全模式只能由用户通过全局 IPC 设置，渲染层不得在单次 fs/shell 调用中覆写 `securityMode` 进行升权；preload、shell IPC、security 模块均实际忽略 per-call `securityMode` 覆盖
- **bash 安全黑名单统一**：提取到共享 JSON 文件（`blocked-patterns.json`），渲染侧与主进程加载同一源，消除两处维护不一致问题
- **补充 Windows 特定危险命令**：`del /f /s`、`reg delete HKLM`、`bcdedit safeboot` 等
- **黑名单正则容错**：无效正则跳过并警告，不阻塞模块加载
- **`security.cjs` 路径修复**：使用 `__dirname` 相对路径，确保 Electron asar 打包后正确加载
- **`ai_review` 模式语义调整**：低风险与中风险操作默认通过，仅对高风险操作做额外判断；工具风险分类扩充，覆盖 `search_files` / `memory` / `knowledge` / `render_widget` / `schedule_task` / `copy_file` / `move_file` 等工具
- **删除路径与命令安全判断增强**：对工作目录内删除与非危险 shell 命令直接放行，其余场景保留额外审查
- **恢复为“AI 审批”语义**：`ai_review` 不再强调“只有拒绝权”，而是要求在每次工具执行前给出明确的允许 / 拒绝结论，并统一输出结构化 `allow + reason`
- **拒绝理由必须具体化**：AI 审批提示词新增硬约束，拒绝时必须点明具体命令片段、目标路径、删除范围、外传对象或权限动作，避免“存在风险”“高危操作”这类通用套话
- **本地拒绝原因兜底增强**：`readonly` / `safe` 模式下的删除与命令拦截现在会返回带具体路径、命令和副作用说明的理由；即便 AI 返回了空泛拒绝语句，也会在本地重写为可定位的具体原因

### 🔧 优化

#### 结构化输出解析

- **新增 `structured-output` 解析模块**：统一处理 JSON、代码块包裹输出及标签式退化输出
- **标题生成器容错**：不再强依赖严格 JSON，支持从 `title` / `标题` 等键值中提取结果
- **工具权限 AI 审查结果解析增强**：支持 JSON 候选、宽松布尔值和 labeled value 退化格式
- **模型输出不稳定容错**：减少因格式轻微偏差导致的失败
- **JSON-like 修复能力补强**：新增对 bare key、单引号、中文冒号、尾逗号等近似 JSON 输出的修复，提升轻量模型结构化响应的可解析率
- **标题生成 prompt 收紧**：标题生成器改为强约束只输出单个 JSON 对象，并补充调试标签与详细日志，便于定位标题生成失败来源
- **权限审批解析扩展**：同时识别 `allow` / `deny` 双语义、自由文本 verdict 和退化 reason，并在拒绝理由泛化时做本地具体化补全

#### 工具装配优化

- **工具装配懒加载**：Computer Use / Browser Use / Audio / Image 工具按运行时可用性条件装配，未连接或不配置时不再全量注册，减少约 52% 无效 token 消耗

#### 记忆与知识库

- **长期记忆语义冲突检测**：写入前检索相似记忆，检测中英文否定词矛盾，返回冲突警告
- **长期记忆结构化写入**：新增 `structured` 参数支持 key-value 键值对，写入时序列化并可在检索时解析展示
- **冲突检测阈值优化**：预搜索阈值从 `0.75` 降至 `0.5`，确保矛盾候选能进入检测函数
- **记忆检索阈值优化**：默认检索阈值降至 `0.5`，减少「未找到」误报；`createMemory` 显式设置 `dedupeThreshold=0.95`
- **知识库检索增强**：新增 `filePath` 和 `dateRange` 过滤参数；缓存从 FIFO 改为精确 LRU

#### LLM 调用重构

- **从流式 API 切换到非流式 API + 重试机制**：更稳定的响应方式
- **统一 `callLlm` 封装**：简化各调用方

#### 工具模块组织结构

- **`web-fetch` 三文件合并为两文件**：`web-fetch-extraction.ts` + `web-fetch-structured.ts` → `web-fetch-core.ts`，减少模块层级（3 层 → 2 层）
- **`index.ts` 导入组织优化**：按功能类别分组（系统信息 / Computer Use / Browser Use / 任务管理 / 网络工具 / 文件操作 / 办公创作 / 知识库 / 长期记忆 / 图片 / 音频 / 开发 / MCP / 技能 / 用户交互 / 团队协作）

#### 其他改进

- **i18n 健壮性**：React 19 兼容性处理，避免 Suspense 边界问题
- **`writeJsonFile` 原子写入**：唯一临时文件名避免并发竞态
- **`cn` 函数语义优化**：`clsx(inputs)` → `clsx(...inputs)`
- **错误处理重构**：error 与 content 分离
- **工具函数提取与去重**：`clampNumber`、`cosineSimilarity`
- **`widget-craft` 内置技能升级为规范型 Skill**：不再依赖内置 HTML 模板，改为提供设计令牌、安全契约、交互桥和模式判断四类参考文档，引导 AI 直接生成更一致的 `widget_code`
- **Widget 宿主视觉统一**：补充 `--widget-*` 主题令牌覆盖、深浅色按钮与表单样式，并收紧 `<hr>` 段间距，减少聊天流中的视觉跳跃
