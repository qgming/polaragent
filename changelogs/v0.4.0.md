# 更新日志

## [0.4.0] - 2026-06-29

### 🎉 新功能

#### 技能引用与上下文展示

- **消息内持续展示技能引用**：对话消息会像文件、图片等引用资源一样保留已选择技能，便于回看本轮对话使用了哪些技能上下文
- **技能上下文更完整**：用户显式选择技能时，除了完整 `SKILL.md` 内容，还会把技能子目录名称与路径拼接到上下文中，让模型知道可继续读取哪些 references、examples、assets 等子文件
- **技能工具增强**：内置技能读取工具支持返回技能目录树，并可读取 references、examples 等子目录文件，方便模型按需渐进式加载技能资料

#### 询问用户工具重构

- **三种交互形态**：`ask_user` 工具统一为 `input`、`single`、`multiple` 三种模式，不再保留旧版 `text` 兼容逻辑
- **选项模式固定补充自定义输入**：单选和多选模式会自动在最后追加“其他 / 补充输入”选项，既能收敛选择，也能让用户补充未覆盖的信息
- **问题支持 Markdown**：询问用户弹窗的问题区域支持 Markdown 渲染，列表、加粗、代码片段等说明更清晰
- **结构化返回更稳定**：输入、普通选项、自定义输入分开返回，便于模型继续可靠处理用户反馈

### 🔧 优化

#### 自动标题生成

- **统一使用默认路由模型**：会话标题生成改为始终使用模型设置里的默认路由模型，不再跟随当前助手或团队 leader 的锁定模型
- **标题生成更稳定**：增加生成中去重，避免并发重复生成；失败后不再永久锁死，后续消息完成时可继续重试
- **标题生成更高效**：只截取早期关键消息生成标题，降低 token 使用；仅在模型明确不支持 JSON 模式时才降级重试，避免无意义的二次调用
- **清空会话后可重新命名**：修复普通会话清空后 `autoTitled` 未重置，导致后续新对话无法自动生成标题的问题

#### 左侧会话列表体验

- **标题宽度恢复**：普通对话、团队行、团队会话子项的右侧操作按钮改为浮层布局，未 hover/focus 时不再占用标题宽度
- **运行状态保留**：会话运行中的 spinner 仍常驻显示，hover/focus 时更多按钮再临时浮出，兼顾状态感知和标题可读性
- **键盘可访问性改进**：更多按钮在 focus-within 状态下同样可见，键盘操作不会丢失入口

#### 团队与消息上下文

- **团队提示同步新协议**：团队成员上下文里的 `ask_user` 使用说明同步为 `input`、`single`、`multiple` 三态，避免继续引导旧协议
- **技能引用持久化解析**：会话消息解析与存储支持技能引用，重新打开历史会话时仍能看到对应技能来源
