# 更新日志

## [0.4.3] - 2026-07-02

### 🎉 新功能

#### 项目（Project）—— 对话分组与专属提示词

- **项目分组**：侧边栏新增「项目」Tab，可将相关对话归类到同一项目下，告别杂乱的对话列表
- **项目专属提示词**：每个项目可设置独立的系统提示词，该项目下所有对话都会自动注入，无需每次手动填写
- **项目共享工作目录**：项目内的所有对话共享同一个工作目录，切换目录时自动同步到项目配置，确保团队成员和会话之间目录一致
- **项目 CRUD**：支持新建、编辑、删除项目，以及清空项目内全部会话
- **会话级助手切换**：每个对话可独立选择使用的助手，不再受全局默认助手的限制。切换助手仅影响当前对话，其他会话不受影响
- **助手持久化**：会话级助手 ID 现在会持久化到 JSONL，重启后不会丢失

### 🐛 Bug 修复

- **修复项目配置并发更新丢失数据**：`updateProject` 存在经典的 lost-update 竞态——两个并行更新会读到相同的旧快照导致先到达的更新被覆盖。现在改为在 `set` 回调内合并更新，确保始终基于最新状态
- **修复删除项目时 harness 内存泄漏**：`handleDeleteProject` 仅调用 `abortAgentThread` 而未调用 `resetAgent`（即 `disposeThread`），导致被删除会话的 harness 引用残留在内存中。现已补上 `resetAgent` 调用
- **修复创建会话时助手 ID 不持久化**：`createThread` 将 `agentId` 写入内存但从未调用 `setSessionAgentId` 持久化到 JSONL，导致重启后对话回退到全局默认助手。现已补上持久化调用
- **修复快速切换线程时项目工作目录回退失败**：`restoreThreadMonitor` 在 `await` 之后重新读取 store，若用户快速切换线程可能读到错误状态。现在在函数开头同步捕获 thread 对象，避免异步后状态漂移
- **修复批量清空/删除项目时磁盘写入失败静默吞掉**：批量操作改用 `Promise.allSettled` 等待全部磁盘操作完成，失败时记录日志，避免内存与磁盘不一致导致重启后旧消息"复活"
- **修复更新项目工作目录时磁盘写入失败静默吞掉**：项目会话切换目录时的 `updateProject` 调用增加了 `.catch()` 错误日志
- **修复重命名会话后项目归属可能在索引重建时丢失**：`renameConversation` 调用 `upsertTitleIndex` 时未传递 `projectId`，若索引条目尚不存在则项目归属丢失。现已补传

### 🔧 优化

- **侧边栏三 Tab 布局**：从原来的「对话 / 团队」两 Tab 扩展为「对话 / 项目 / 团队」三 Tab，宽度由 220px 调整为 240px
- **团队列表展开动画**：团队会话子列表展开/收起增加高度动画（AnimatePresence + motion.div），交互更流畅
- **团队行新增「+」按钮**：在团队行右上角直接添加「新建对话」按钮，无需展开后再点新建
- **项目列表默认展开当前项目**：进入属于某项目的对话时，侧边栏自动展开该项目
